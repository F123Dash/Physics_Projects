# Turbulence-CNN
### Physics-Informed Super-Resolution U-Net for 2-D Lid-Driven Cavity Flow

A deep learning pipeline that reconstructs high-resolution (64×64) velocity, pressure, and vorticity fields from 4× coarse (16×16) inputs, trained on incompressible Navier-Stokes solutions across twelve Reynolds number regimes (Re = 100–3200). An auxiliary classifier identifies the Reynolds number regime from the bottleneck features with 100% accuracy.

---

The CNN outperforms bicubic interpolation by **642% in SSIM** on OOD flows. At Re=2800 (never seen during training), the prediction recovers the Kolmogorov −5/3 spectral scaling to within **0.4%**.

## Physical Background

### Governing Equations

The solver integrates the 2-D incompressible Navier-Stokes equations:

$$\frac{\partial \mathbf{u}}{\partial t} + (\mathbf{u}\cdot\nabla)\mathbf{u} = -\frac{1}{\rho}\nabla p + \nu\nabla^2\mathbf{u}, \qquad \nabla\cdot\mathbf{u} = 0$$

Non-dimensionalised with lid velocity $U=1$ m/s and side length $L=1$ m, so $\nu = 1/Re$.

### Numerical Method

The solver uses **Chorin's fractional-step projection method** (1968):

1. **Intermediate velocity** — advection (first-order upwind) and diffusion (central differences), no pressure correction
2. **Pressure Poisson** — $\nabla^2 p = (\rho/\Delta t)\,\nabla\cdot\mathbf{u}^*$, solved by SOR with $\omega=1.7$
3. **Projection** — $\mathbf{u}^{n+1} = \mathbf{u}^* - (\Delta t/\rho)\nabla p$

Stability is enforced by an adaptive time step satisfying both the CFL ($\Delta t < \alpha\,\Delta x / |\mathbf{u}|_\text{max}$) and the viscous ($\Delta t < \alpha\,\Delta x^2 / 4\nu$) constraints, with safety factor $\alpha=0.2$.

### Super-Resolution as an Inverse Problem

Downsampling by factor 4 (block averaging) is a low-pass filter that retains only $6.25\%$ of the spatial frequency content. The inverse problem — recovering the fine field from the coarse field — is ill-posed: infinitely many fine fields produce the same coarse observation. The CNN resolves this non-uniqueness by learning the statistical prior of cavity flow solutions from 8909 training snapshots.

The boundary layer at the right wall has thickness $\delta \sim L/Re^{1/2}$, which is **sub-pixel at 16×16 for all Re in the training set**. This feature is physically irrecoverable from the input — the CNN infers it from large-scale context.

## Pipeline

The pipeline has four stages. Each stage must complete before the next begins.

```
ns_solver.py  →  dataset.py  →  train.py  →  evaluate.py
```

---

## Stage 0 — Generate Snapshots

```bash
python solver/ns_solver.py [--seed SEED] [--validate_ghia]
```

Runs the NS solver for all 12 Re values and saves snapshots to `snapshots/Re_*/`.

| Argument | Type | Default | Description |
|---|---|---|---|
| `--seed` | int | random | RNG seed for reproducibility |
| `--validate_ghia` | flag | off | Run N=128 Ghia validation instead of data generation. Solves Re=100 and Re=1000 on a 128×128 grid and prints quantitative benchmark errors. Does not save snapshots. |

**Output**: `snapshots/Re_{re}/snap_{n:05d}.npy` — shape `(4, 64, 64)` float32, channels `[u, v, p, ω]`.

**Expected terminal output per Re:**
```
Config: Re=600, N=64×64, nu=0.00167, t_end=80.0, t_start_save=40.0, save_every=1000
  Converged at t=31.4, step=31400
Done — saved 150 snapshots to snapshots/Re_600/
```

**Notes:**
- Snapshots are saved only after `t_start_save = 0.50 × t_end` to discard transients (0.40 × t_end for Re > 1500)
- `save_every` is Re-dependent: 2000 steps for Re≤400, 1000 for Re≤1000, 500 for Re>1000
- High-Re flows (Re > 1500) that converge before `t_start_save` still save 150 post-convergence snapshots

## Stage 1 — Build Dataset

```bash
python data/dataset.py [--max_per_class MAX]
```

Loads all snapshots, computes normalisation statistics, applies the outlier filter, and writes `train-data/stats.npy`.

| Argument | Type | Default | Description |
|---|---|---|---|
| `--max_per_class` | int or `"auto"` | None | Cap snapshots per Re class. `"auto"` sets the cap to 4× the median class count, reducing worst-case imbalance while keeping > 80% of data. Pass an integer for a hard cap. `None` keeps all data. |

**Output**: `train-data/stats.npy` — dict with keys `mean` and `std`, shape `(4,)` each.

**Expected terminal output:**
```
Total snapshots loaded: 8909
  ω_mean = -0.8566 (negative = clockwise = correct ✓)
Normalisation statistics:
  [0] u-velocity       mean=+0.0117  std=0.1896
  [1] v-velocity       mean=+0.0023  std=0.1238
  [2] pressure         mean=-0.0000  std=0.0353
  [3] vorticity ω      mean=-0.8566  std=3.3823
Split: train=7128  val=1334  test=447
Class weights: [4.949 4.949 0.819 0.463 0.465 0.534 0.639 0.901 1.213 3.437 4.949 4.949]
```

**Notes:**
- The outlier filter checks only velocity channels (u=0, v=1) at threshold 10σ ≈ 1.9 m/s. Vorticity is not filtered — boundary layers reach 7–9σ and are physically valid.
- `ω_mean < 0` is asserted as a physics sanity check: clockwise primary vortex → net negative domain vorticity.
- Class weights use inverse-frequency weighting: $w_c = N_\text{total} / (12 \times n_c)$

## Stage 2 — Train

```bash
python train/train.py \
    --data_dir  snapshots \
    --out_dir   runs/exp_01 \
    --epochs    150 \
    [--resume   PATH] \
    [--seed     SEED] \
    [--lr       LR] \
    [--lr_min   LR_MIN] \
    [--lambda_div  LAMBDA_DIV] \
    [--lambda_vort LAMBDA_VORT] \
    [--lambda_cls  LAMBDA_CLS] \
    [--base_ch  BASE_CH] \
    [--dropout_p DROPOUT_P] \
    [--batch_size BATCH_SIZE] \
    [--grad_clip  GRAD_CLIP] \
    [--num_workers NUM_WORKERS]
```

| Argument | Type | Default | Description |
|---|---|---|---|
| `--data_dir` | str | `snapshots` | Path to the directory containing `Re_*/` snapshot subdirectories |
| `--out_dir` | str | `runs/exp_01` | Output directory for checkpoints, plots, and `history.npy` |
| `--epochs` | int | 150 | Total training epochs. The cosine LR schedule spans exactly this many epochs. |
| `--resume` | str | None | Path to a checkpoint (e.g. `runs/exp_01/last_model.pt`) to resume from. Restores model weights, optimizer state, and scheduler state. Training continues from `epoch+1`. |
| `--seed` | int | random | RNG seed for reproducibility across NumPy, PyTorch, and CUDA |
| `--lr` | float | 1e-3 | Initial learning rate (peak of cosine schedule) |
| `--lr_min` | float | 1e-5 | Minimum learning rate (floor of cosine schedule at epoch `epochs`) |
| `--lambda_div` | float | 0.01 | Weight on the physical divergence loss $\lambda_\text{div}\|\partial u/\partial x + \partial v/\partial y\|_2^2$. Increasing to 0.1 enforces stronger incompressibility at the cost of ~0.01 SSIM. |
| `--lambda_vort` | float | 0.001 | Weight on the vorticity consistency loss. **Do not increase above 0.001** — the boundary layer vorticity MSE is physically irrecoverable (floor ~112) and at 0.01 consumed 70% of the optimizer budget. |
| `--lambda_cls` | float | 0.01 | Weight on the Re-regime classifier cross-entropy loss. The classifier saturates at 100% accuracy around epoch 40; this term's primary function is to force Re-ordered encoder representations. |
| `--base_ch` | int | 64 | Base channel width of the U-Net. The four encoder levels have `[64, 128, 256, 512]` channels. Reduce to 32 to cut parameters by 4×. |
| `--dropout_p` | float | 0.1 | Dropout probability in encoder blocks. Applied during training only. |
| `--batch_size` | int | 16 | Batch size. Increase to 32 if GPU memory allows — reduces training time by ~30%. |
| `--grad_clip` | float | 1.0 | Maximum L2 norm of the gradient. Prevents exploding gradients during the first few epochs when loss is high. Set to 0 to disable. |
| `--num_workers` | int | 0 | DataLoader worker processes. Set to 4 on Linux systems with fast storage. Leave at 0 on Windows. |

**Outputs in `--out_dir`:**
- `best_model.pt` — checkpoint at minimum `val_loss`
- `best_ssim_model.pt` — checkpoint at maximum `val_ssim` (**use this for evaluation**)
- `last_model.pt` — checkpoint after every epoch (for resuming)
- `history.npy` — dict with all metrics per epoch (train/val loss, recon, div, vort, ssim, acc, lr)
- `training_curves.png` — 2×3 dashboard: total loss, recon, divergence, vorticity, accuracy, SSIM
- `prediction_ep{N:03d}.png` — prediction panels saved every 10 epochs
- `confusion_matrix.png` — 12×12 Re-regime confusion matrix on test set

## Reproducing the Full Pipeline

```bash
# Stage 0: generate all snapshots (~2 hours on CPU)
python solver/ns_solver.py --seed 42

# Stage 1: build dataset
python data/dataset.py

# Stage 2: train for 150 epochs (~6 hours on GPU)
python train/train.py \
    --data_dir  snapshots \
    --out_dir   runs/exp_01 \
    --epochs    150 \
    --seed      42

# Stage 3: OOD evaluation
python train/evaluate.py \
    --checkpoint  runs/exp_01/best_ssim_model.pt \
    --stats_path  train-data/stats.npy \
    --out_dir     runs/eval_ood \
    --n_samples   100 \
    --seed        42
```

To resume a crashed training run:
```bash
python train/train.py \
    --data_dir  snapshots \
    --out_dir   runs/exp_01 \
    --epochs    150 \
    --resume    runs/exp_01/last_model.pt
```

To reuse existing OOD snapshots (skips re-generation):
```bash
python train/evaluate.py \
    --checkpoint   runs/exp_01/best_ssim_model.pt \
    --stats_path   train-data/stats.npy \
    --out_dir      runs/eval_ood_v2 \
    --ood_snap_dir runs/eval_ood/ood_snapshots
```

## Model Architecture

```
Input (4, 64, 64)  [u, v, p, ω]
    │
    ├── enc1: Conv(4→64)   + Dropout(0.1)  → (64, 64, 64)
    ├── enc2: Pool + Conv(64→128)          → (128, 32, 32)
    ├── enc3: Pool + Conv(128→256)         → (256, 16, 16)
    │
    └── bottleneck: Pool + Conv(256→512)   → (512, 8, 8)
                        │
                        ├── RegimeClassifier → logits (12,)
                        │
    ┌── dec1: Up + Cat(skip3) + Conv → (256, 16, 16)
    ├── dec2: Up + Cat(skip2) + Conv → (128, 32, 32)
    ├── dec3: Up + Cat(skip1) + Conv → (64, 64, 64)
    │
Output (4, 64, 64)  [û, v̂, p̂, ω̂]
```

**Parameters**: 7,318,800 total  
**Upsampling**: Bilinear interpolation + 1×1 Conv (avoids checkerboard artefacts)  
**Activations**: LeakyReLU(0.2)  
**Normalisation**: InstanceNorm2d in encoder blocks

## Loss Function

$$\mathcal{L} = \mathcal{L}_\text{recon} + \lambda_\text{div}\mathcal{L}_\text{div} + \lambda_\text{vort}\mathcal{L}_\text{vort} + \lambda_\text{cls}\mathcal{L}_\text{cls}$$

| Term | Formula | Weight | Purpose |
|---|---|---|---|
| Reconstruction | $\sum_c w_c\|\hat{f}_c - f_c\|_2^2$, weights=[3,3,2,2] | 1.0 | Pixel-wise fidelity |
| Divergence | $\|\sigma_u\partial\hat{u}/\partial x + \sigma_v\partial\hat{v}/\partial y\|_2^2$ | 0.01 | Incompressibility |
| Vorticity | $\|\hat{\omega} - (\partial\hat{v}/\partial x - \partial\hat{u}/\partial y)\|_2^2$ | 0.001 | Internal consistency |
| Classifier | Weighted cross-entropy on Re logits | 0.01 | Re-aware representations |

The divergence term uses **physical units** (scaled by $\sigma_u$, $\sigma_v$) to ensure the incompressibility penalty is symmetric across u and v.

## Known Limitations

| Limitation | Physical cause |
|---|---|
| Right-wall boundary layer not reconstructed | Sub-pixel feature at 16×16 — information-theoretically irrecoverable |
| Secondary corner vortices absent | Scale < 3 px at 64×64 — invisible in 16×16 input |
| Physical divergence = 0.033 m/s per m | CNN has no hard incompressibility constraint |
| Spectral slope shallower than −5/3 at Re=1800 | Large-scale cavity vortex dominates low-k spectrum |
